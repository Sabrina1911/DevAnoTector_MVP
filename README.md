## **DevAnoTector MVP — Agile Project**

### **AI-Powered Smart Implant & Wearable Monitoring System**

This repository contains the minimum viable product (MVP) of DevAnoTector, a simulation-driven anomaly and safety monitoring tool designed for medical implants and wearables.

*It includes:*
   - A React role-based frontend (Clinician + Engineer panels)
   - A Node.js backend with rule-based scoring
   - A Simulation engine for What-If tuning
   - A Risk visualization component (RiskChart)
   - Telemetry factor handling (misalignment, temperature, load, charge rate)

### **Project Overview**

DevAnoTector is designed to help clinicians and engineers:

*Clinicians*
   - View simplified traffic-light safety statuses
   - See reasons behind device alerts
   - Understand contributing factors

**Engineers**
   - Run What-If simulations
   - Adjust coil misalignment, charge rate, temperature, load
   - Compare real-world telemetry with rule-based predictions
   - View risk contributions via RiskChart
All simulation results are generated by the backend’s Rules Engine + Explainability Engine.

### **Key Features**

*Role-Based Frontend (React + TypeScript)*
   - Secure role routing (Clinician / Engineer)
   - Modular panel design (ClinicianRolePanel, EngineerRolePanel)
   - Shared RolePanel wrapper component
   - RiskChart.tsx for visualizing risk contributions

*Backend (Node.js + Express)*
   - /patients → fetch PHI/research patients
   - /simulate → compute score, status, factors
   - /simulate/sweep → generate angle/temperature/rate sweeps

*Simulation Engine*

Factors include:
   - Misalignment
   - Charge rate
   - Temperature
   - Load

Each factor contributes to an overall severity score.

**Explainability Engine**
Returns:

{
  score: number,
  status: "GREEN" | "AMBER" | "RED",
  reasons: [...],
  factors: {
    misalign,
    rate,
    temp,
    load
  }
}

**Architecture Summary**
The DevAnoTector MVP uses:
   - Frontend: React (Vite) + TypeScript
   - Backend: Node.js + Express
   - Data: Simple JSON / SQLite (optional)
   - Security: Basic role simulation (no login yet)

**Available API Endpoints**

GET /patients?role=clinician

Returns PHI-safe subset.

GET /patients?role=engineer

Returns research dataset.

POST /simulate

   {
  "coilOffsetDeg": 10,
  "chargeRateC": 1.3,
  "tempC": 39,
  "load_mA": 200
}

Returns score, status, factors.

POST /simulate/sweep

Runs bulk simulation.
